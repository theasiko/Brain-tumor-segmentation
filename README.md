## Setup

```bash
pip install -r requirements.txt
```

## Train

```bash
python train.py --epochs 10 --batch-size 8 --img-size 128
```

Model weights will be saved to `checkpoints/best_model.pt`.

## Evaluate

```bash
python evaluate.py --checkpoint checkpoints/best_model.pt
```

## Visualize predictions

```bash
python visualize.py --checkpoint checkpoints/best_model.pt --split test --indices 3 15 40
```

## Notes
- Images are converted to grayscale.
- CLAHE preprocessing is applied before resizing.
- Masks are built from polygon segmentations.
- Training uses BCEWithLogitsLoss + Dice loss.

1. Введение

Анализ медицинских изображений является одной из ключевых задач современных интеллектуальных систем в здравоохранении. Магнитно-резонансная томография (МРТ) широко используется для диагностики заболеваний головного мозга, включая опухолевые образования. Однако интерпретация МРТ-снимков требует высокой квалификации врача-радиолога и значительных временных затрат.

В связи с этим актуальной задачей является разработка интеллектуальных систем, способных автоматически анализировать медицинские изображения и выделять патологические области. В данной работе рассматривается задача семантической сегментации опухолей головного мозга на МРТ-снимках с использованием методов глубокого обучения.

Целью работы является построение и анализ простой, интерпретируемой модели сегментации, демонстрирующей полный pipeline обработки изображений: от загрузки данных до визуализации результатов, а также показ возможности использования модели в интеллектуальной медицинской системе.

3. Предметная область и постановка задачи

2.1 Предметная область

В качестве предметной области выбрана медицина, а именно:

•	анализ медицинских изображений,

•	сегментация опухолей головного мозга на МРТ-снимках.

МРТ является неинвазивным методом визуализации, позволяющим получать детализированные изображения мягких тканей, что делает его особенно ценным для диагностики опухолей мозга.

2.2 Формулировка задачи

Задача формулируется как задача семантической сегментации:

Для каждого пикселя входного МРТ-изображения необходимо определить, принадлежит ли он опухоли или фону.

Таким образом, результатом работы модели является бинарная маска, совпадающая по размеру с исходным изображением, где:

•	1 — пиксель относится к опухоли,

•	0 — пиксель относится к фону.

4. Используемый датасет

В работе используется датасет “Brain Tumor Image Dataset for Semantic Segmentation”, загруженный с платформы Hugging Face. Ссылка: dwb2023/brain-tumor-image-dataset-semantic-segmentation · Datasets at Hugging Face

# Brain-tumor-segmentation

В данной работе рассматривается задача семантической сегментации опухолей головного мозга на МРТ-снимках с использованием методов глубокого обучения.

3.1 Характеристики датасета

Датасет содержит:

•	МРТ-снимки головного мозга в формате RGB;

•	аннотации опухолей в виде полигональных сегментаций (COCO-формат);

•	разбиение на обучающую, валидационную и тестовую выборки.

Размер датасета:

•	обучающая выборка: ~1500 изображений,

•	валидационная выборка: ~400 изображений,

•	тестовая выборка: ~200 изображений.

Общий размер датасета - 162 Мб.

3.2 Особенности датасета

•	сильный дисбаланс классов (опухоль занимает малую часть изображения);

•	сегментация представлена в виде координат полигонов, требующих преобразования в бинарную маску;

•	высокая вариативность формы и размера опухолей.

4. Формализованная модель обработки изображений (Pipeline)

Процесс обработки изображений в данной работе можно представить в виде следующего pipeline:

Загрузка данных → Предобработка → Выделение признаков → Сегментация → Визуализация результатов

5. Этапы обработки изображений

5.1 Загрузка данных

Данные загружаются с помощью библиотеки datasets из Hugging Face.

Каждый элемент датасета содержит:

•	изображение МРТ;

•	информацию о сегментации опухоли в виде полигонов.

5.2 Предобработка изображений

Предобработка включает следующие этапы:

1.	Преобразование изображения в оттенки серого

Используется, поскольку МРТ не содержит цветовой информации, а интенсивность пикселей важнее цветовых каналов.

2.	Изменение размера изображения

Все изображения приводятся к единому размеру (например, 128×128), что необходимо для пакетной обработки нейронной сетью.

3.	Нормализация

Значения пикселей масштабируются в диапазон [0, 1], что ускоряет и стабилизирует обучение модели.

Contrast Limited Adaptive Histogram Equalization (CLAHE) является модификацией классического алгоритма гистограммного выравнивания и широко используется в задачах медицинской обработки изображений. В отличие от глобального histogram equalization, CLAHE выполняет выравнивание контраста локально, разбивая изображение на небольшие регионы (tiles). Для каждого региона строится и нормализуется собственная гистограмма яркости. Чтобы предотвратить чрезмерное усиление шума, вводится ограничение (clip limit) на максимальное значение гистограммы, после чего результаты локального выравнивания интерполируются между соседними регионами.

Применение CLAHE позволяет:

1.	Улучшить визуальную различимость опухолевых тканей, особенно на границе с окружающими здоровыми структурами;

2.	Снизить влияние неинформативных областей фона, что облегчает задачу модели;

3.	Стабилизировать распределение интенсивностей, улучшая сходимость нейронной сети при обучении.

За счёт усиления локального контраста модель получает более информативное представление признаков, что особенно важно для архитектур типа U-Net, использующих свёрточные фильтры для извлечения локальных паттернов.

5.3 Подготовка масок сегментации

Аннотации сегментации представлены в виде координат полигонов. Для обучения нейронной сети они преобразуются в бинарные маски, где:

•	пиксели внутри полигона → 1,

•	остальные пиксели → 0.

Этот шаг является критически важным, так как ошибка в построении маски приводит к некорректному обучению модели.

6. Модель и методы

6.1 Архитектура модели

Для решения задачи семантической сегментации опухолей головного мозга была использована сверточная нейронная сеть архитектуры U-Net, адаптированная для обработки двумерных медицинских изображений (2D MRI-срезов).

Архитектура U-Net является стандартом де-факто в медицинской сегментации благодаря способности эффективно объединять:

•	локальные пространственные признаки (низкоуровневые детали),

•	контекстную информацию (глобальную структуру изображения).

В качестве базового блока сети используется модуль DoubleConv, состоящий из двух последовательных свёрточных слоёв с ядром 3×3, нормализацией по батчу и функцией активации ReLU. Данный блок обеспечивает эффективное извлечение локальных признаков и стабильное обучение модели.

Кодирующая часть сети последовательно уменьшает пространственное разрешение изображения с помощью Max Pooling, увеличивая количество каналов и формируя более абстрактные представления. В центральной части (bottleneck) модель извлекает высокоуровневые контекстные признаки.

Декодирующая часть восстанавливает пространственное разрешение с использованием транспонированных свёрток. Skip connections между кодирующими и декодирующими слоями позволяют сохранить пространственные детали и повысить точность локализации опухолей.

Финальный слой представляет собой свёртку 1×1, формирующую карту вероятностей принадлежности каждого пикселя к опухолевой области.

Выбор U-Net обусловлен её высокой эффективностью при работе с ограниченными медицинскими датасетами и хорошей интерпретируемостью результатов сегментации.

6.2 Функция потерь

Для обучения используется комбинация:

•	Binary Cross-Entropy Loss

•	Dice Loss

Dice Loss особенно эффективна в задачах сегментации при сильном дисбалансе классов, так как напрямую оптимизирует пересечение между предсказанной и истинной маской.

6.3 Процесс обучения

Обучение модели осуществляется в несколько эпох с использованием:

•	оптимизатора Adam;

•	пакетной обработки данных (mini-batch training).

На ранних этапах обучения модель склонна предсказывать фон, однако по мере обучения начинает локализовать опухоль, что является типичным поведением для медицинской сегментации.

7. Результаты и оценка качества

Для оценки качества сегментации использовались следующие метрики:

•	Dice coefficient

•	Intersection over Union (IoU)

•	Pixel Accuracy

Пример результатов на тестовой выборке:

•	Dice ≈ 0.54

•	IoU ≈ 0.43

•	Pixel Accuracy ≈ 0.97

Высокая Pixel Accuracy объясняется доминированием фонового класса, поэтому основной упор делается на Dice и IoU.

Разработанная модель может быть использована как часть интеллектуальной медицинской системы:

1.	МРТ-снимок загружается в систему.

2.	Модель автоматически выделяет область подозрения на опухоль.

3.	Результат визуализируется и передаётся врачу.

4.	Врач использует результат как decision support tool, а не как окончательный диагноз.

Такая система может:

•	ускорить анализ изображений;

•	снизить нагрузку на специалистов;

•	повысить точность и воспроизводимость диагностики.

8. Заключение

В данной работе была разработана и проанализирована модель семантической сегментации опухолей головного мозга на МРТ-снимках. Был реализован полный pipeline обработки изображений, включающий загрузку данных, предобработку, обучение модели, оценку качества и визуализацию результатов.
Несмотря на ограниченный размер датасета и простоту модели, результаты демонстрируют работоспособность подхода и его применимость в интеллектуальных медицинских системах. Работа показывает, что даже базовые модели глубокого обучения способны решать практические задачи анализа медицинских изображений при корректной постановке задачи и интерпретации результатов.
